# 马尔代夫选岛助手 — 设计文档

## 概述

一个帮助用户筛选马尔代夫度假岛屿的交互式网页工具。支持多维度筛选、银河摄影条件智能计算，以及快捷推荐。

## 技术架构

### 系统架构

```
GitHub 仓库
├── data.json           ← 岛屿数据（每天自动更新）
├── index.html          ← 前端筛选网页
├── scraper/            ← 数据抓取脚本
└── .github/workflows/  ← 每天自动运行抓取

Vercel（免费部署）→ 全球 CDN → 用户浏览器
```

### 数据流

1. **GitHub Action** 每天定时触发 → 运行爬虫脚本 → 更新 `data.json`
2. 用户打开网页 → 从 GitHub Pages / Vercel 获取最新 `data.json` → 渲染筛选结果

### 部署

- 使用 **Vercel**（免费版）部署前端
- 源码托管在 **GitHub** 公开仓库
- `.superpowers/` 加入 `.gitignore`

### 银河摄影计算逻辑

根据用户输入的**出行日期**，前端自动计算：

| 因素 | 计算方式 | 来源 |
|------|---------|------|
| 🌙 **月相** | 日期 → 月龄 → 月出月落时间 | 天文算法（内置） |
| 🌌 **银心可见** | 日期 + 纬度 → 银心升起/落下时间 | 天文算法（内置） |
| 💡 **光污染** | Bortle 等级 → 可见度影响 | 岛屿数据字段 |
| 🌅 **日落时间** | 日期 + 纬度 → 民用黄昏结束 | 天文算法（内置） |

**综合评分规则：**
- **优**：银心在夜间升起 + 月龄 < 5 或月已落 + Bortle ≤ 3 + 岛上灯光控制良好
- **良**：银心在夜间升起 + 月龄或光污染有一项不理想
- **差**：银心不在夜间升起 / 满月 / Bortle ≥ 6

### 筛选维度

- **卡片**：白色圆角卡片，柔和阴影
- **筛选标签**：浅色背景圆角标签，选中时变主题色

## 数据更新机制

### GitHub Action 工作流

- **触发**：每天 UTC 00:00 定时运行 + 手动触发
- **任务**：运行 Python 爬虫 → 更新 data.json → 提交到仓库
- **爬取来源**：Agoda / Booking.com / TripAdvisor 等公开页面（仅提取价格、评分等结构化信息）
- **失败处理**：如果某个来源不可用，保留上次数据并在 data.json 中标记 `"数据来源": "上次缓存"`
- **防滥用**：设置合理请求间隔，遵守 robots.txt

### 初始数据

首次发布时由 AI 助手根据公开资料整理 20-30 个热门马尔代夫岛屿的初始数据（含经纬度、光污染等级等），后续由爬虫自动更新价格和评分。
